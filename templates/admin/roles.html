Role mapping

Map Keystone role names to application capabilities. A user who has any of the listed roles (in any project, via any group, or via domain inheritance) is granted the capability cluster-wide. Local administrators always hold every capability and bypass this mapping.

Role names are stored lowercased. Removing the last role mapped to manage_config is refused for non-local-admin callers — only a local administrator can perform that revocation.
